# 悟空租车会员升级数据收集页

一个无外部依赖的 Node.js 小应用，包含：

- 微信端可分享的移动端填写页：手机号、校友卡号
- Supabase 数据库存储，未配置 Supabase 时本地回退到 `data/submissions.jsonl`
- 管理页预览数据
- 下载 Excel 可打开的 `.xls` 文件，字段为：序号、校友id、手机号、时间
- 备用 CSV 下载

## 本地运行

```bash
cd wukong-member-collector
ADMIN_TOKEN=your-secret-token PORT=3000 npm start
```

用户填写页：

```text
http://localhost:3000/
```

后台下载页：

```text
http://localhost:3000/admin.html?token=your-secret-token
```

## Supabase 建表

在 Supabase 项目的 SQL Editor 执行：

```sql
create table if not exists public.submissions (
  id bigint generated by default as identity primary key,
  alumni_id text not null,
  phone text not null,
  created_at timestamptz not null default now(),
  user_agent text,
  ip text
);

alter table public.submissions enable row level security;
```

后端使用 `SUPABASE_SERVICE_ROLE_KEY` 访问数据库。这个 key 只能放在 Render 环境变量里，不要写进 GitHub，不要放到前端页面。

## Render 免费部署

使用仓库里的 `render.yaml` 创建 Blueprint。Render 会要求填写三个环境变量：

| 名称 | 说明 |
| --- | --- |
| `ADMIN_TOKEN` | 后台访问密码 |
| `SUPABASE_URL` | Supabase Project URL |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase `service_role` secret key |

部署完成后，将 Render 分配的 HTTPS 首页链接发到微信即可收集数据。

## 部署和微信分享

把这个应用部署到有公网 HTTPS 域名的服务器后，将首页链接发到微信即可收集数据。

生产环境建议：

- 必须设置 `ADMIN_TOKEN`，不要使用默认值 `change-me`
- 正式收集数据时必须配置 Supabase，不要依赖免费实例的本地文件
- 使用 HTTPS 域名，微信内打开体验更稳定
- 定期从 Supabase 备份 submissions 表

## 环境变量

| 名称 | 说明 | 示例 |
| --- | --- | --- |
| `PORT` | 服务监听端口，云平台通常会自动注入 | `3000` |
| `ADMIN_TOKEN` | 后台访问密码 | `a-long-random-secret` |
| `SUPABASE_URL` | Supabase 项目 URL | `https://xxxx.supabase.co` |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase 服务端密钥 | `eyJ...` |
| `DATA_DIR` | 本地 JSONL 回退存储目录，仅本地开发需要 | `/var/data` |
